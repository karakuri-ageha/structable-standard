## 第6章：比較表

小説形式と台本形式の共通点と差異を整理する。

### 6.1 共通点

| 要素      | 内容                           |
| ------- | ---------------------------- |
| 核心原則    | 声に出した言葉だけがセリフテーブルに入る         |
| セリフ形式   | 話者｜セリフ のテーブルが基本構造            |
| 目的      | 話者の明確化、認知負荷の軽減、視覚的な読みやすさの向上  |
| 心の声の扱い  | セリフ列には入れない                   |
| 話者列     | 発話者の名前を記載。必要に応じて状況を付記可       |
| 場面転換    | 場面が変わるたびに挿入する                |
| コードブロック | 両形式で使用可能（小説：孤立した断片、台本：モノローグ） |

### 6.2 差異

| 要素              | 小説                     | 台本                   |
| --------------- | ---------------------- | -------------------- |
| セリフテーブル         | 話者｜セリフ（2列）             | 話者｜セリフ｜演技指示（3列）      |
| 地の文             | あり（`>`引用ブロック、【話者】明記）   | なし                   |
| 心の声（`>>`）       | あり（思考先行型）              | なし                   |
| 心の声（横配置）        | あり（セリフと同時）             | なし                   |
| 心の声（縦配置）        | あり（セリフの後）              | なし                   |
| 心の声（非テーブル）      | 地の文に含める                | モノローグ（コードブロック）で処理    |
| 記憶の断片（`>>>`）    | あり                     | なし                   |
| 孤立した断片（コードブロック） | 帰属先のない断片               | モノローグ（演出テーブルで話者指定）   |
| 場面転換            | `---` + 太字見出し          | 場所｜時間 のテーブル          |
| 演出              | なし                     | 演出｜内容 のテーブル          |
| ナレーション          | 地の文に溶け込む               | 専用テーブル               |
| モノローグ           | 心の声（>>・横配置・縦配置）や地の文で処理 | コードブロック（演出テーブルで話者指定） |
| ふりがな（`<ruby>`）  | 使用可                    | 使用可                  |

### 6.3 使用テーブル・記法一覧

|テーブル・記法|小説|台本|
|---|---|---|
|セリフ（話者｜セリフ）|✓|✓|
|セリフ（話者｜セリフ｜演技指示）|✗|✓|
|心の声・横配置（話者｜セリフ｜心の声）|✓|✗|
|心の声・縦配置（セリフテーブル内でドッキング）|✓|✗|
|心の声・引用ブロック（`>> 【話者】`）|✓|✗|
|地の文（`> 【話者】`）|✓|✗|
|記憶の断片（`>>>`）|✓|✗|
|孤立した断片 / モノローグ（コードブロック）|✓|✓|
|場面転換（`---` + 太字見出し）|✓|✗|
|場面転換（場所｜時間）|✗|✓|
|演出（演出｜内容）|✗|✓|
|ナレーション|✗|✓|
|ふりがな（`<ruby>`）|✓|✓|
